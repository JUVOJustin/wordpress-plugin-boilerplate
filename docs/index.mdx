Demo Plugin gives you a structured starting point for building modern WordPress plugins. Use this landing page to choose the workflow you need, understand the moving parts, and jump directly to the relevant guide.

## Start Here

<Steps>
1. Create your plugin from the boilerplate.

   ```bash title="Terminal"
   composer create-project juvo/wordpress-plugin-boilerplate
   ```

2. Start the local WordPress environment.

   ```bash title="Terminal"
   npm run env:start
   ```

3. Pick the guide that matches your next task.

   - Building blocks: [create-blocks.mdx](./create-blocks.mdx)
   - Managing assets: [bundeling.mdx](./bundeling.mdx)
   - Running tests: [testing.mdx](./testing.mdx)
   - Preparing translations: [i18n.mdx](./i18n.mdx)
</Steps>

## Repo At A Glance

<FileTree>
- demo-plugin.php Main bootstrap file
- src/
  - Abilities/ WordPress abilities and categories
  - API/ REST API-specific code
  - Blocks/ Gutenberg block implementations
  - CLI/ WP-CLI commands
  - Integrations/ Third-party and platform integrations
- resources/
  - admin/ Admin-side scripts and styles
  - frontend/ Frontend scripts and styles
- tests/
  - php/ PHPUnit application tests
- docs/ This documentation site
</FileTree>

## What This Boilerplate Optimizes For

- clear separation between bootstrap code, plugin logic, assets, docs, and tests
- reproducible local development with `wp-env`
- production-ready quality gates for PHP, JavaScript, styles, and application tests
- documented extension points for blocks, abilities, translations, and integrations
- AI-friendly project instructions without replacing human-readable docs
