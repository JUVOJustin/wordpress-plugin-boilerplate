Use this guide to start writing application tests quickly in **Demo Plugin**.  
Focus here is on authoring real plugin tests (assertions, factories, hook behavior), not on low-level PHPUnit wiring.

## Start your first application test

<Steps>
1. Start the test environment.

```bash title="Terminal"
npm run env:start
```

2. Create your first test class in `tests/php`.

```php title="tests/php/ExampleTest.php"
<?php

class ExampleTest extends WP_UnitTestCase {
	public function test_plugin_bootstraps_core_class(): void {
		$this->assertTrue( class_exists( \Demo_Plugin\Demo_Plugin::class ) );
	}
}
```

3. Run tests.

```bash title="Terminal"
npm run test:php
```

4. Add more test classes in `tests/php` as features grow (`phpunit.xml.dist` already points there).
</Steps>

## Common test class examples

### Test behavior that depends on WordPress data

```php title="tests/php/PostMetaBehaviorTest.php"
<?php

class PostMetaBehaviorTest extends WP_UnitTestCase {
	public function test_can_store_and_read_post_meta(): void {
		$post_id = self::factory()->post->create();
		update_post_meta( $post_id, '_demo_flag', 'yes' );

		$this->assertSame( 'yes', get_post_meta( $post_id, '_demo_flag', true ) );
	}
}
```

### Test hooks or filters from your plugin

```php title="tests/php/FilterRegistrationTest.php"
<?php

class FilterRegistrationTest extends WP_UnitTestCase {
	public function test_demo_filter_modifies_value(): void {
		$result = apply_filters( 'demo_plugin/example_value', 'before' );
		$this->assertNotSame( 'before', $result );
	}
}
```

## Why this guide uses wp-env

`wp-env` gives Demo Plugin a ready WordPress + PHPUnit runtime with test config already wired, so you can focus on writing assertions instead of maintaining local test infrastructure.
